Support for :visited styling varies across browsers due to security concerns.

  1. In IE10 adjacent sibling selector doesn't work with pseudo-class in case of E:active F.
  2. IE6 does not properly support combinations of pseudo classes like :link, :active and :visited
  3. IE8-11 does not update an element's :hover status when scrolling without moving the pointer.
  4. Safari 5.1 and Android browsers do not support the adjacent selector if the adjacent element is a "nav" element.
  5. :first-child fails in IE7 if the first child is a comment.
  6. IE7 doesn't support all pseudo classes (like :focus) or pseudo elements (like :before and :after)
